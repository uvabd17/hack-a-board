# Security Policy

## Supported Versions
- `main` branch: supported

## Reporting a Vulnerability
Please report security issues privately by opening a private security advisory in GitHub or contacting the maintainers directly.

Include:
- affected endpoint or component
- impact summary
- reproducible steps
- proof-of-concept (if available)

Do not publicly disclose vulnerabilities before maintainers acknowledge and remediate.

## Scope
In-scope:
- authentication/authorization bypass
- sensitive data exposure
- injection vulnerabilities
- token/session flaws
- abuse/rate-limit bypass

Out-of-scope:
- social engineering
- denial-of-service attacks
- spam reports without reproduction steps
